International Debt Analysis Platform

Python Pandas NumPy Matplotlib Seaborn SQL Power BI Jupyter

A comprehensive data analytics platform developed as the final year project for BCA at BBDU. The platform analyses international debt data across multiple countries and time periods using publicly available World Bank and IMF datasets. It performs advanced statistical analysis and generates interactive visualisations to uncover patterns in global debt dynamics, debt-to-GDP ratios, and creditor composition.

📌 Key Features & Highlights

  • 🔍 Multi-country debt trend analysis with time-series visualisation (line charts, area charts)
  • 📈 Debt-to-GDP ratio benchmarking across 50+ economies using World Bank indicators
  • 🗂️ Creditor composition breakdown: bilateral vs multilateral vs commercial creditors
  • 📊 Interactive Power BI dashboard for stakeholder-level reporting
  • 🧹 End-to-end data cleaning pipeline: missing value imputation, normalisation, feature engineering
  • 📉 Correlation analysis between debt levels and macroeconomic indicators (GDP growth, inflation)
  • 🌍 Country-level risk profiling based on debt sustainability indicators

🛠️ Technical Approach

Data was sourced from the World Bank Open Data API and IMF public datasets. Python (Pandas, NumPy) was used for data wrangling and preprocessing. EDA was performed using Matplotlib and Seaborn, and a Power BI dashboard was built for executive-level visualisation. SQL queries were used for data aggregation and filtering.

ShubhVivah – Wedding Vendor SQL Analysis

MySQL SQL Window Functions Recursive CTEs Views Triggers Stored Procedures Python

An end-to-end MySQL portfolio project built around a genuinely uncommon angle: how Shubh Muhurat (auspicious wedding dates) drive demand surges and dynamic pricing in India's wedding vendor industry. Models a fictional marketplace ShubhVivah connecting couples with vendors (photographers, caterers, decorators, venues, DJs) across 15 Indian cities on a 2,000-row synthetic dataset.

📌 Key Features & Highlights

  • 📐 10-table relational schema with primary/foreign keys & 4 performance indexes
  • 📊 25 graded business queries: foundations → window functions → recursive CTEs → optimisation
  • 📈 Muhurat demand-surge analysis using AVG() OVER, RANK() OVER, LAG() OVER
  • 🔁 Recursive CTEs to traverse referral networks & calculate customer lifetime value
  • 🛠️ vendor_dashboard & muhurat_demand_surge views for BI reporting
  • ⚙️ book_vendor stored procedure enforcing double-booking prevention
  • 🔔 log_price_change trigger for automatic price-history audit logging
  • 🐍 Python seed-data generator script to recreate/expand the 2,000-row dataset anytime

🛠️ Technical Approach

Database designed from scratch in MySQL 8.0+ to leverage window functions and recursive CTEs. All 10 tables (cities, vendor_categories, muhurat_dates, vendors, customers, bookings, price_history, reviews, referrals, payments) are seeded with deterministic synthetic data generated by a pure-Python script. The analytics layer (views, procedures, triggers) is designed to simulate the reporting infrastructure a real marketplace ops team would use.
